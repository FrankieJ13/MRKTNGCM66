# MRKTNGCM66

Дашборд маркетинговой эффективности CM66 под закрытую Google-таблицу.

## Что внутри

- KPI-обзор: расходы, заявки, квалы, визиты, продажи, цена продажи.
- Воронка: заявки → квалы → визиты → продажи в кредит.
- Рейтинги каналов и городов по расходам и результату.
- Дневная динамика по ключевым метрикам.
- Контроль качества источника и готовность к подключению Google Sheets.
- Детектор таблиц по ключевым ячейкам, а не по фиксированным координатам.

## Источник данных

Проект не хранит данные в репозитории. Дашборд читает закрытую Google-таблицу через `src/config.js`:

```js
window.MRKTNG_CONFIG = {
  googleSheetUrl: "https://docs.google.com/spreadsheets/d/...",
  sheetNames: {
    aggregate: "Все вместе_Факт"
  },
  appsScriptProxyUrl: "",
  preferProxy: false
};
```

Для закрытой таблицы прямой `fetch` может не пройти из-за CORS или Google-авторизации. В таком случае разверните Apps Script-прокси от имени аккаунта с доступом и укажите его URL в `appsScriptProxyUrl`, затем поставьте `preferProxy: true`.

Инструкция и готовый код прокси: `docs/GOOGLE_SHEETS.md`.

## Локальный запуск

Откройте `index.html` или поднимите локальный сервер:

```bash
python3 -m http.server 8000
```

## Рекомендуемая структура Google Sheets

Основной агрегированный лист должен сохранять шапку как в экспортированном файле:

- `Города`
- `Расходы`
- `Заявки`
- `Цена заявки`
- `Не целевые`
- `% не целевых`
- `Квалы`
- `Цена квала`
- `Визиты`
- `% из квала в визит`
- `Цена визита`
- `Продажи в кредит`
- `Цена продажи в кредит`
- `Наличка`
- `Обмен`

Текущий парсер умеет читать повторяющиеся блоки в листах по шапке `Города / Расходы / Заявки`. Для более надёжного боевого режима всё равно лучше держать отдельный нормализованный лист `dashboard_data` с колонками:

`period`, `city`, `channel`, `view`, `spend`, `leads`, `qualified`, `visits`, `credit_sales`, `cash_sales`, `bad_leads`, `gross_income`, `roi`.
